# Project Overview

## Course Context

AI Risk Detector was created for the Agile Project Management course as a group project about Agile, Scrum, DevOps, MVP delivery, and responsible AI.

The assignment focuses on applying project management concepts in a realistic product context. The application is intentionally lightweight so that the project can focus on product thinking, Scrum evidence, DevOps reasoning, and critical reflection.

## Product Purpose

AI Risk Detector helps Agile and DevOps teams analyse software tasks, requirements, or user stories before execution.

The assistant supports earlier discussion about:

- Unclear backlog items
- Missing acceptance criteria
- Delivery risks
- Testing needs
- Release readiness
- Mitigation actions

The goal is to improve planning conversations and reduce late discovery of problems.

## Main Users

The product is intended for:

- Product Owners
- Scrum Masters
- Developers
- QA or testing roles
- DevOps-oriented team members

## Value Proposition

The assistant turns a natural language work item into a structured risk report.

The report can help the team decide whether a backlog item is clear enough, what could go wrong, what tests may be needed, and what mitigation actions should be discussed before implementation.

## Project Scope

This project is not a production-ready enterprise platform. It is an academic MVP used to demonstrate how Agile Project Management concepts can be applied to a product idea.

Detailed backlog, Scrum, DevOps, MVP, and reflection content is documented in the other files in this folder.
