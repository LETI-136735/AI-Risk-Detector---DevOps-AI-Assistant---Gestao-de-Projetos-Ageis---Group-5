# DevOps and MVP

## Purpose

This document explains how the project connects DevOps thinking with the MVP of AI Risk Detector.

## DevOps Alignment

The project connects to DevOps through:

- Version control with GitHub
- Repository structure
- Dependency management through `requirements.txt`
- Automated workflow evidence in GitHub Actions
- Testing and quality considerations
- Release readiness thinking
- Feedback loops between planning, implementation, testing, and improvement

## Product Connection to DevOps

AI Risk Detector supports DevOps-oriented teams by helping them identify risks earlier.

The assistant can support:

- Shift-left quality
- Suggested tests
- Mitigation planning
- Release readiness discussion
- Deployment risk awareness
- Continuous improvement

## MVP Description

The MVP demonstrates the core value of the product without unnecessary complexity.

The user provides a software task, requirement, or user story. The assistant returns a structured risk analysis containing:

- Risk level
- Potential risks
- Impact explanation
- Mitigation recommendations
- Suggested tests
- Possible acceptance criteria
- Backlog improvement suggestions

## In Scope

The MVP includes:

- Basic input of a work item
- Risk analysis output
- Simple risk classification
- Suggested tests and mitigation actions
- Human-readable structured response

## Out of Scope

The MVP does not include:

- Full Jira API integration
- Production monitoring
- Enterprise AI governance
- Automated release management
- Advanced analytics dashboard
